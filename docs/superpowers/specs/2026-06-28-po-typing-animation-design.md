# Po 打字動畫設計（Po Typing Animation）

> 日期：2026-06-28　狀態：待使用者審閱
> 目標：讓 Po 從「浮在鍵盤上方比讚」進化成「**真的在打那台鍵盤 / 用滑鼠**」，
> 像 Bongo Cat。手會跟著打字的位置移動；打字模式與滑鼠模式互斥（不同時出現）。

---

## 1. 總覽

- Po 維持在**上方、面向使用者**。
- 桌面（鍵盤＋滑鼠）**整個旋轉 180°**，呈現「從 Po 那一側看過去」的視角
  （空白鍵那排在上、靠近 Po；數字列在下、靠近觀看者；滑鼠在左上）。
- Po 的外觀由 **7 張完整影格圖**驅動。程式依「目前是哪種輸入」挑影格，
  並負責「按下去的下壓動作」與「對應鍵 / 滑鼠鍵亮起」。
- **鍵盤亮鍵、滑鼠左右鍵/滾輪反應**已經做好（現況），這個功能是疊在其上。

## 2. 視窗與座標（沿用現況，不變）

- 視窗畫布：**350 × 249**（`config.CANVAS_WIDTH/HEIGHT`）。
- 鍵盤 chassis `(12,138,278,99)`、滑鼠 `(302,181,36,56)`，繪製時對「桌面中心
  `(175, 187.5)`」套用 **180° 旋轉**。
- Po 影格圖以視窗原點 `(0,0)` 對齊整張畫布（見 §4 的註冊方式）。

## 3. 手 → 鍵盤位置對應（4 區）

- 依**實體鍵的 x 座標**（鍵盤本地座標、旋轉前）把鍵盤分成 4 個直向區：
  - 區 1 = 左外、區 2 = 左內、區 3 = 右內、區 4 = 右外。
- 左手負責 區 1–2，右手負責 區 3–4（由「哪張影格的哪隻手壓下」表現）。
- 每次按鍵：
  1. 由該鍵中心 x → 判定區 → 選對應打字影格。
  2. 該鍵在**旋轉後的位置**亮起（沿用現有亮鍵系統）。
  3. 影格做一個小幅「下壓」位移（程式控制，美術不用畫上下兩版）。
- 連續打字會在 4 張打字影格之間切換，跟著你打的位置跑。
  影格切換做輕微節流（最短間隔 ~50ms）避免快速打字時閃爍；但**亮鍵每次都觸發**。

## 4. 七張影格（美術規格）

每張都是一張**完整的 Po 圖**（頭＋身體＋雙手在該姿勢），規格一致以便無縫替換：

| 檔名（`assets/frames/`） | 內容 |
|---|---|
| `po_idle.png` | 待機：雙手輕放/微抬在鍵盤上，沒有特別壓下 |
| `po_type_l1.png` | 左手壓在**左外**區 |
| `po_type_l2.png` | 左手壓在**左內**區 |
| `po_type_r1.png` | 右手壓在**右內**區 |
| `po_type_r2.png` | 右手壓在**右外**區 |
| `po_mouse.png` | 右手放在**滑鼠**上（左手休息），沒按 |
| `po_mouse_down.png` | 右手**按下滑鼠**（與 `po_mouse` 同位置，手指略壓） |

規格：
- 尺寸 **350 × 249（與視窗同大）**、RGBA、透明背景，只畫 Po（含手臂、手）。
- 繪製時參考提供的**註冊範本** `assets/frames/_template.png`（內含旋轉後的鍵盤、
  4 區分界、滑鼠、Po 頭部圈位、各區手部落點），確保手落在正確區、頭對齊位置。
- Po 影格會被疊在「程式畫的桌面（鍵盤＋滑鼠）」之上，原點對齊即完美套準。
- 維持 Po 的橘黑風格與既有比例。

> 範本由程式產生（見 §7），使用者拿範本當底層描，畫完隱藏底層匯出即可。

## 5. 模式切換（鍵盤 / 滑鼠互斥）

狀態：`IDLE`、`TYPING(zone)`、`MOUSING`。
- 按鍵 → 進 `TYPING`，顯示對應區影格。
- 滑鼠鍵 / 滾輪 → 進 `MOUSING`，顯示 `po_mouse`；左鍵/右鍵/中鍵或滾動時，
  該滑鼠部位亮起；按下時短暫顯示 `po_mouse_down`。
- **最後發生的輸入獲勝**，另一種的影格不會同時出現。
- 距離最後一次輸入超過 `REACTION_DECAY_MS`（預設 800ms）→ 回 `po_idle`。
- 直接對 Po 本體 hover/拖曳/點擊：維持目前影格 + 一個小 pop，不切換影格
  （保持單純；未來可再加專屬反應）。

## 6. 遮罩 / 點擊穿透

- Po 遮罩改由**目前影格的 alpha**產生（整張畫布），隨下壓位移平移。
- 最終遮罩 = Po 影格遮罩 ∪ 旋轉後的桌面（鍵盤＋滑鼠）footprint。
- 透明處一律穿透，沿用現有 `setMask` 機制。

## 7. 工程做法：先佔位、後換圖（可立即動工）

1. 先做好**影格驅動引擎**（影格載入、依模式/區選影格、下壓位移、模式切換、遮罩）。
2. 真圖還沒到之前，用**程式產生的佔位影格**（例如沿用現有姿勢或簡單剪影 + 手形）
   讓整套動畫先能跑、能調手感。
3. 程式啟動時自動偵測 `assets/frames/po_*.png`：**檔案齊全就用真圖，否則用佔位**。
4. 同時產生並輸出 `assets/frames/_template.png` 給美術描圖。
5. 使用者畫好 7 張丟進 `assets/frames/`，重開即上線，無需改程式。

## 8. 影響的檔案

- 新增 `src/po_frames.py`：影格集載入、模式/區 → 影格、佔位生成、範本輸出。
- 新增 `src/scene_layout.py`（或併入 config）：桌面 180° 旋轉的 transform 與
  「實體鍵 x → 區 → 影格」對應、各區旋轉後的落點。
- 修改 `pet_window.py`：Po 改用影格系統繪製；桌面套旋轉；輸入路由到模式/區；
  遮罩改用影格 alpha。
- 修改 `keyboard.py`：輸出「鍵 → 區」資訊；繪製與遮罩支援旋轉。
- 修改 `config.py`：旋轉中心、區邊界、影格路徑、模式時間參數。
- `input_listener.py`：已提供 token / button / scroll，無需大改。

## 9. 測試

- 離線 render 測試：7 種影格 × 旋轉後桌面，各出一張 PNG 目視。
- 單元測試：`鍵 token → 區 → 影格` 對應正確（抽樣多顆鍵）。
- 遮罩 sanity：Po 影格中心 in-mask、鍵盤/滑鼠 in-mask、透明角 out。
- 手感：連續打字的影格切換不閃爍、模式切換乾淨、停手回待機。

## 10. 非目標（這次不做）

- 不做「每顆鍵一張影格」的精準對位（4 區足夠）。
- 不做打字＋滑鼠同時出現。
- 不處理燒進原圖的文字、不上 Live2D、不做對話泡泡（屬 PLAN.md 其他階段）。
